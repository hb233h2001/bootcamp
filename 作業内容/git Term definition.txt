- リポジトリ - ソースコードのバージョン管理のためにGitを使用するリポジトリ。
                Gitリポジトリは、プロジェクトのすべてのファイルと変更履歴を保存し、
                これによって協業とバージョン管理を容易にできるようにする。 
                リポジトリはローカルリポジトリとリモートリポジトリに分けられる。
- リモートリポジトリ - インターネットやネットワーク経由でアクセスできる共用リポジトリ。
- ローカルリポジトリ - コンピュータに保存されるリポジトリ.
- ブランチ - 独立したタスクのためにコードベースを分離する機能。 ブランチを使用すると、
                メインコードラインを維持しながらも様々な機能開発、バグ修正、実験などを同時に行うことが可能。 
                これにより、開発過程での衝突を最小限に抑え、効率的な協業ができる。
- リモートブランチ -  リモートストレージにあるブランチ
- ローカルブランチ - ユーザーのコンピュータに存在するブランチ。 リモートストレージから分離して使用可能。 
                    リモートストレージに上書き可能。インポート可能。
- デフォルトブランチ - 基本的に設定されているブランチ。 変更可能
- ワークツリー - 一つのレポジトリで複数のブランチを作業領域という。 ブランチ=ワークツリーではないが、開発のために作業を分ける場合、
                これをワークツリーという。 gitディレクトリを除くすべての領域。
- インデックス / ステージ - インデックス = staging area. git addをしたファイルが存在する領域。
                ここにあるファイルだけがコミットに反映される。
